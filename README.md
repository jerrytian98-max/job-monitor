# 招聘监测系统

一个自动监测招聘网站并发送邮件通知的Python程序，提供Web可视化界面。可以定时扫描目标公司的招聘信息，当发现符合关键词的职位时，自动发送邮件通知。

## 功能特性

- 🌐 **Web管理界面**：美观的可视化界面，方便配置管理
- 📊 **实时状态监控**：实时显示监测状态、发现职位数量、运行时长等
- 🔔 **多网站支持**：支持BOSS直聘、拉勾网、猎聘网
- 🏢 **公司监控**：自定义目标公司列表
- 🔍 **智能匹配**：根据关键词、城市等条件筛选职位
- 🚫 **去重过滤**：自动过滤重复职位，只通知新职位
- ⏰ **定时监测**：自动定时监测，无需手动操作
- 📧 **邮件通知**：美观的HTML格式邮件通知
- ⚙️ **灵活配置**：支持多种配置选项，包括排除关键词等

## 安装步骤

### 1. 安装Python依赖

确保已安装Python 3.7+，然后运行：

```bash
pip install -r requirements.txt
```

### 2. 安装Flask（Web界面依赖）

```bash
pip install flask
```

### 2. 配置系统

编辑 `config.yaml` 文件，根据您的需求修改配置：

#### 配置目标公司

```yaml
target_companies:
  - "阿里巴巴"
  - "腾讯"
  - "字节跳动"
  - "华为"
  - "美团"
```

#### 配置职位关键词

```yaml
job_keywords:
  - "Python"
  - "Java"
  - "前端工程师"
  - "后端开发"
```

#### 配置城市筛选

```yaml
cities:
  - "北京"
  - "上海"
  - "深圳"
  - "杭州"
```

#### 配置邮箱通知

这是最重要的配置，需要填写真实的邮箱信息：

```yaml
email:
  sender: "your_email@gmail.com"           # 发件邮箱
  auth_code: "your_auth_code"              # 邮件授权码（重要！）
  receiver: "your_email@gmail.com"         # 接收通知的邮箱
  smtp_server: "smtp.gmail.com"            # SMTP服务器
  smtp_port: 587                           # SMTP端口
```

#### 配置监测间隔

```yaml
check_interval: 2  # 每2小时检查一次
```

#### 配置排除关键词

```yaml
exclude_keywords:
  - "实习"
  - "兼职"
```

## 获取邮箱授权码

不同的邮箱获取授权码的方式不同：

### Gmail

1. 登录Google账户
2. 进入账户设置 -> 安全性
3. 开启"两步验证"
4. 生成"应用专用密码"
5. 使用生成的16位密码作为 `auth_code`

### QQ邮箱

1. 登录QQ邮箱网页版
2. 点击"设置" -> "账户"
3. 找到"POP3/IMAP/SMTP/Exchange/CardDAV/CalDAV服务"
4. 开启"POP3/SMTP服务"
5. 按提示发送短信获取授权码
6. 使用获取的授权码

### 163邮箱

1. 登录163邮箱网页版
2. 点击"设置" -> "POP3/SMTP/IMAP"
3. 开启"IMAP/SMTP服务"
4. 按提示获取授权码
5. 使用获取的授权码

## 使用方法

### 方式一：Web界面（推荐）

提供可视化Web界面，方便配置和管理。

#### 1. 启动Web服务

```bash
python app.py
```

启动后会显示访问地址，默认为：`http://127.0.0.1:5000`

#### 2. 访问Web界面

在浏览器中打开 `http://127.0.0.1:5000`，您将看到：

**监测状态面板**
- 实时显示监测运行状态
- 累计发现职位数量
- 今日新职位数量
- 上次检查时间和运行时长
- 启动/停止监测按钮

**配置表单**
- 目标公司列表（支持动态添加/删除）
- 职位关键词（支持动态添加/删除）
- 城市筛选（可选）
- 排除关键词（实习、兼职等）
- 监测网站URL（支持多个）
- 邮箱配置
- 监测间隔设置

**快捷操作**
- 刷新配置
- 测试邮件发送
- 查看日志
- 清除职位缓存

#### 3. 配置监测参数

在Web界面中：

1. **添加目标公司**：在"目标公司"输入框中输入公司名称，按回车添加
2. **设置职位关键词**：在"职位关键词"输入框中输入关键词，按回车添加
3. **选择城市**：（可选）添加想要的城市
4. **添加监测网站**：在"监测网站"中添加招聘网站URL
5. **配置邮箱**：填写发件邮箱、授权码、接收邮箱等信息
6. **设置监测间隔**：选择检查频率（建议2-4小时）

#### 4. 保存配置

点击"保存配置"按钮保存所有设置。

#### 5. 测试检查

点击"测试检查"按钮，验证配置是否正确（测试模式会在控制台输出，不会发送邮件）。

#### 6. 启动监测

配置验证通过后，点击"启动监测"按钮开始自动监测。

监测将持续运行，定期检查招聘网站并发送邮件通知。

### 方式二：命令行

如果习惯使用命令行，也可以使用以下方式：

#### 测试模式

测试模式会在控制台输出结果，不会发送邮件，用于验证配置是否正确：

```bash
python main.py --test --once
```

#### 单次检查

运行一次完整的检查和通知：

```bash
python main.py --once
```

#### 持续监测（自动循环）

程序会持续运行，定时检查并发送通知：

```bash
python main.py
```

#### 自定义配置文件

如果使用不同的配置文件：

```bash
python main.py --config my_config.yaml
```

## 工作原理

1. **爬取职位**：从配置的招聘网站爬取目标公司的职位信息
2. **匹配过滤**：根据公司、关键词、城市等条件筛选职位
3. **去重检测**：与本地缓存对比，识别新职位
4. **发送通知**：发现新职位时发送邮件通知
5. **定时循环**：按照设定的时间间隔重复上述流程

## 注意事项

### 反爬虫限制

实际使用中，招聘网站通常有反爬虫机制。当前版本使用模拟数据，实际部署时需要：

1. 添加请求延迟（代码已实现）
2. 使用代理IP池
3. 模拟真实浏览器行为
4. 考虑使用Selenium等工具
5. 遵守网站的robots.txt规定

### 邮箱安全

- 不要在代码或配置文件中直接保存邮箱密码
- 使用授权码而非邮箱密码
- 不要将包含敏感信息的配置文件提交到公开仓库
- 建议使用专门用于通知的邮箱账号

### 频率控制

- 监测间隔不要太短，建议至少2小时
- 避免对同一网站频繁请求
- 尊重网站的使用条款

## 常见问题

### Q: 如何启动Web界面？

**A:** 运行 `python app.py`，然后在浏览器中打开 `http://127.0.0.1:5000`

### Q: Web界面无法访问？

**A:** 检查以下几点：
1. 确认已安装Flask：`pip install flask`
2. 检查端口5000是否被占用
3. 查看控制台是否有错误信息
4. 尝试使用 `python app.py` 启动

### Q: 为什么没有收到邮件？

**A:** 检查以下几点：
1. 邮箱配置是否正确（sender、auth_code、receiver）
2. 授权码是否正确且有效
3. SMTP服务器地址和端口是否正确
4. 防火墙是否阻止了程序的网络访问
5. 在Web界面中点击"测试检查"验证配置
6. 或使用命令行测试：`python main.py --test --once`

### Q: 如何添加新的招聘网站？

**A:** 
1. 在Web界面的"监测网站"部分添加网站URL
2. 或编辑 `config.yaml` 添加网站URL
3. 当前版本使用模拟数据，实际使用时需要在 `scraper.py` 中实现真实的爬取逻辑

### Q: 如何修改邮件样式？

**A:** 编辑 `notifier.py` 中的 `_create_email_content()` 方法，修改HTML模板。

### Q: Web界面和命令行可以同时使用吗？

**A:** 不建议。Web界面和命令行使用相同的配置文件和缓存文件，同时使用可能导致冲突。建议只使用其中一种方式。

### Q: 程序可以部署到服务器吗？

**A:** 可以。建议：
1. 使用 `nohup python main.py &` 后台运行
2. 使用systemd或supervisor管理进程
3. 添加日志记录和错误处理
4. 定期检查 `jobs_cache.json` 状态
